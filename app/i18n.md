# Readest 多语言 (i18n) 支持指南

本文档介绍在 `readest-app` 项目中如何添加和管理多国语言支持。

## 1. 语言配置文件
项目支持的语言列表维护在根目录的 `i18n-langs.json` 文件中。当前支持包含 `en`, `zh-CN`, `zh-TW`, `de`, `ja` 等多种语言。如果需要新增语言种类，需要先在此文件中添加对应的语言代码（如 `fr`）。

## 2. 翻译文件位置
所有的具体翻译文本存储在 `public/locales/` 目录下。该目录下按照语言代码划分子目录：
- `public/locales/en/translation.json` (英文)
- `public/locales/zh-CN/translation.json` (简体中文)
- `public/locales/zh-TW/translation.json` (繁体中文)

**注意：** Readest 的 `i18n` 配置禁用了对象嵌套（`keySeparator: false`），因此在 JSON 文件中**不支持**嵌套结构。必须使用扁平的键值对结构，例如：
```json
{
  "Home": "首页",
  "Your Library": "书库"
}
```

## 3. 在代码中使用
项目中已经封装了翻译 Hook `useTranslation`，位于 `src/hooks/useTranslation.ts`。

**使用示例：**
```tsx
import { useTranslation } from '@/hooks/useTranslation';

const MyComponent = () => {
  const _ = useTranslation();

  // 普通文本翻译
  const title = _('Your Library');

  // 带有命名空间的键名
  const homeText = _('appHeader.home');

  return <div>{title}: {homeText}</div>;
};
```

**带变量的翻译：**
如果在翻译文本中有变量（例如：`"{{count}} pages left"`）：
```tsx
const pagesText = _('{{count}} pages left', { count: 5 });
```

## 4. 参考旧版翻译
旧版 MyReader 前端的翻译资源位于 `reference/mybooks_frontend/locales/` 中，如 `en.json` 和 `zh.json`。它们使用了嵌套的 JSON 结构。当向 Readest 移植时，需要将嵌套结构展平后（如 `appHeader` 下的 `home` 变成 `appHeader.home`）再添加到 `public/locales/{lang}/translation.json` 中。
